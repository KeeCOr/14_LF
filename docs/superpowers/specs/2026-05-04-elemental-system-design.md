# 속성 에너지 시스템 설계 (2026-05-04)

## 개요

슬롯머신이 카드를 뽑는 구조에서, 덱이 손패를 공급하고 슬롯머신은 속성 에너지를 생성하는 구조로 전환한다. 플레이어는 속성 에너지를 소모해 손패의 카드를 배치한다.

---

## 1. 속성 에너지 (ElementalEnergy)

3가지 속성을 별도 카운터로 관리한다.

| 속성 | 심볼 |
|------|------|
| 화 (Fire) | 🔥 |
| 철 (Iron) | ⚔️ |
| 생명 (Life) | 💚 |

최대 보유량: 각 속성별 10개 (조정 가능)

---

## 2. 릴 시스템 (ReelSystem)

### 덱 연동 심볼 풀

덱 8장의 에너지 비용 합산 → 릴 심볼 출현 비율 결정

예시) 덱에 화 요구 합계 8, 철 4, 생명 4 → 화 50% / 철 25% / 생명 25%

### 스핀 결과 에너지 계산

| 릴 결과 | 획득 에너지 |
|---------|------------|
| 전부 다름 (화/철/생명) | 각 1개씩 (총 3) |
| 2개 일치 (화/화/철) | 일치 속성 3 + 나머지 1 (총 4) |
| 3개 일치 (화/화/화) | 해당 속성 6 (3 기본 + 3 보너스) |

슬롯 비용: 행운 1개 소모 (기존 시스템 유지)

---

## 3. 덱 & 손패 시스템

### 덱 구성 규칙
- 8장 고정 (유닛/마법/건물 자유 혼합)
- 동일 카드 최대 2장

### 손패 동작
1. 게임 시작 시 덱 앞 4장 → 손패 지급
2. 손패 4장 모두 소진 시 → 나머지 4장 자동 지급
3. 덱 순환: 전부 소진 시 처음부터 반복

### 배치
- 유닛/건물: 아군 영역 클릭 배치
- 마법: 화면 어디든 시전 가능 (적 기지 포함)
- 비용: 해당 카드의 속성 에너지 소모

## 4. 카드 목록

### 유닛 (10종)

| 유닛 | 에너지 비용 | 특성 | 공중공격 |
|------|------------|------|---------|
| 검사 | 철×1 | 근접, 범용 | ✗ |
| 기사 | 철×2 | 고HP 탱커, 느림 | ✗ |
| 마법사 | 화×2 | 원거리, 범위피해 | ✓ |
| 힐러 | 생명×2 | 아군 회복 | ✗ |
| 궁수 | 화×1 + 철×1 | 원거리, 빠른 공속 | ✓ |
| 행운술사 | 철×1 + 생명×1 | 에너지 자동생성 | ✗ |
| 팔라딘 | 철×2 + 생명×2 | 초고HP, 광역저항 | ✗ |
| 화염술사 | 화×3 | 광역화염, 유리몸 | ✓ |
| 성기사 | 화×1 + 철×2 + 생명×1 | 공격 + 주변 회복 오라 | ✗ |
| 폭풍궁수 | 화×2 + 철×2 | 다중타겟, 고속 | ✓ |

### 마법 (4종)

| 마법 | 에너지 비용 | 효과 |
|------|------------|------|
| 번개화살 | 화×2 | 단일 대상 강력 피해 |
| 포탈폭격 | 화×2 + 철×1 | 범위 피해 |
| 생명의비 | 생명×3 | 아군 전체 회복 |
| 철벽강화 | 철×2 + 생명×1 | 아군 전체 일시 방어력 증가 |

마법은 적 기지(Village)에도 직접 사용 가능하다.

### 건물 — 전투형 (3종)

| 건물 | 에너지 비용 | 효과 | 공중공격 |
|------|------------|------|---------|
| 화염탑 | 화×2 | 원거리 자동 공격 | ✓ |
| 철벽 | 철×1 | 이동 방해 장벽 | ✗ |
| 저격탑 | 화×1 + 철×1 | 장거리 고피해 | ✓ |

### 건물 — 생산형 (5종)

| 건물 | 에너지 비용 | 생산 내용 |
|------|------------|----------|
| 화염로 | 화×1 | 화 에너지 자동 생성/초 |
| 제철소 | 철×1 | 철 에너지 자동 생성/초 |
| 생명의샘 | 생명×1 | 생명 에너지 자동 생성/초 |
| 병영 | 철×2 | 일정 시간마다 검사 자동 소환 |
| 마법진 | 화×2 + 생명×1 | 일정 시간마다 마법사 자동 소환 |

## 6. 기술 구조

### 새 컴포넌트

| 컴포넌트 | 역할 |
|---------|------|
| `ElementalEnergy` | 화/철/생명 카운터 관리, 소모/충전 API |
| `ReelSystem` | 덱 기반 심볼 풀 계산, 스핀 → 에너지 지급 |
| `BuildingController` | 전투/생산 건물 공통 베이스 |
| `BattleBuilding` | 자동 공격 건물 동작 |
| `ProductionBuilding` | 에너지 생성 or 유닛 자동 소환 |

### 수정 컴포넌트

| 파일 | 변경 내용 |
|------|----------|
| `CardData.cs` | `placementCost(int)` 제거 → `ElementalCost` 구조체 추가 |
| `SlotMachineSystem.cs` | 행운 충전 시스템 유지, ReelSystem과 분리 |
| `GameManager.cs` | ElementalEnergy 인스턴스 관리 추가 |
| `UnitController.cs` | `isFlying` 플래그, `canAttackAir` 플래그 추가 |
| `MonsterController.cs` | `isFlying` 플래그, 공중 타겟팅 로직 추가 |
| `ArenaSystem.cs` | 건물 배치 처리 추가 |
| `HandUI.cs` | 속성 에너지 비용 표시 |
| `TestSceneBootstrapper.cs` | 전체 카드 데이터 재작성 |

### 구현 순서

1. `ElementalCost` 구조체 + `CardData` 수정
2. `ElementalEnergy` 시스템 + `GameManager` 연동
3. `ReelSystem` + 릴 UI 수정
4. 덱 순환 딜링 (`DeckSystem` 수정)
5. 건물 시스템 (`BuildingController`, `BattleBuilding`, `ProductionBuilding`)
6. 공중 유닛 플래그 + 타겟팅 로직
7. 몬스터 종류 추가
8. `TestSceneBootstrapper` 카드 데이터 전체 업데이트
