1. 게임 개요

항목내용
장르룰렛 × 타워 디펜스 × 카드 게임
플랫폼PC / 모바일 (Unity 2022 URP 2D)
인원1인 vs AI / 생존 모드 (싱글)
한 판 시간전투: 3분 / 생존: 무제한
수익화F2P + IAP (캐릭터 획득·성장 중심)

핵심 루프

룰렛 자동 회전
  → 행운 소모(릴 1개 정지당 1회)
  → 3릴 모두 정지 → 속성 에너지(불/철/생) 획득
  → 에너지 소모 → 카드 배치 (유닛 / 건물)
  → 유닛이 몬스터 격퇴 → 기지 방어
  → 적 기지 파괴 → 승리
속성 에너지는 불(Fire) · 철(Iron) · 생(Life) 세 종류. 카드마다 요구 속성이 다르며, 룰렛 결과(두 개 같으면 +1, 세 개 같으면 +3)로 누적된다.

2. 게임 모드

모드설명
전투 모드3분 제한 시간 내 AI 기지 파괴. STAGE 시스템으로 난이도 상승
생존 모드무한 웨이브를 최대한 버팀. 웨이브마다 몬스터 수·엘리트 비율 증가
  • 게임 시작 시 덱에서 랜덤 카드 1장 손패에 지급
  • 승패 확정(기지 파괴 / 타임오버) 시 게임 즉시 정지 (Time.timeScale = 0)
  • 결과 화면에서 재시작 버튼 누르면 씬 리로드 + timeScale 복원

3. 씬 레이아웃

x=-7.5        x=-2.5      x=0        x=+2.5       x=+7.5
[플레이어기지]  [적웨이브출구]  [포탈]  [AI웨이브출구]  [AI기지]
  (시안)                  (보라/빨강)                (오렌지)
영역설명
좌측 (x < 0)플레이어 진영. 몬스터가 x=-2.5에서 출현해 기지(-7.5)로 이동

씬은 Unity Editor 씬 파일 없음. TestSceneBootstrapper.cs가 런타임에 모든 오브젝트를 생성한다.

4. 속성 에너지 시스템 핵심

룰렛 결과로 획득되는 자원. 카드 배치 시 소모된다.

속성아이콘UI 색상주로 사용되는 카드
불(Fire)🔥#FF8833화염술사, 드래곤 건물 등 공격형
철(Iron)⚔#99CCFF기사, 방어탑 등 방어형
생(Life)💚#33FF77힐러, 생명 건물 등 회복형

에너지 HUD

  • 화면 우상단에 아이콘 + 숫자로 표시 (예: 🔥 3 ⚔ 1 💚 0)
  • 0일 때 아이콘/숫자가 회색으로 흐려짐
  • 카드 배치 실패(에너지 부족) 시 부족한 속성 레이블이 빨간색으로 4회 점멸

손패 비용 표시

  • 각 카드 버튼에 요구 비용이 속성 색상으로 표시됨
  • 보유 에너지가 부족한 속성은 빨간색(#FF4444)으로 강조
  • 전체적으로 감당 불가능한 카드는 카드 배경이 어둡게 변함

5. 룰렛 시스템 변경됨

5.1 행운(Luck) 충전

항목수치
행운 자동 충전 주기2초마다 +1
시작 시 지급행운 3회
릴 1개 정지 비용행운 1회

5.2 릴 동작 방식

  • 룰렛은 항상 자동으로 회전 (게임 내내 멈추지 않음)
  • 버튼("🔒 릴 N 정지 (xN)")을 누를 때마다 행운 1 소모 → 릴 하나 정지
  • 행운이 없으면 릴을 멈출 수 없음
  • 3개 모두 정지되면 에너지 계산 후 자동으로 다음 스핀 시작

5.3 에너지 계산 규칙

결과조건에너지 보상
올미스3릴 모두 다른 속성각 속성 +1씩 (총 3)
더블2릴 동일 속성해당 속성 +2, 나머지 +1 (보너스 +1 포함)
트리플3릴 모두 동일 속성해당 속성 +6 (보너스 +3 포함)
트리플 달성 시 금색 화면 플래시 연출(ScreenFlash) 발생.

6. 카드 & 손패

  • 손패 슬롯 4개. 항상 가득 찬 상태 유지 — 카드 사용 즉시 덱에서 보충
  • 덱이 비면 더 이상 보충 없음 (현재 덱 = 20장 이상)

유닛어두운 파랑클릭 선택 → 배치 구역 클릭 또는 드래그 → 배치
건물금색 계열유닛과 동일 (배치 구역 내 드랍)
스킬어두운 보라클릭 선택 → 원하는 지점 클릭 → 즉시 발동
버프금색클릭 즉시 발동

11. 유닛 AI

매 프레임:
  1. sightRange 이내에서 가장 가까운 몬스터 탐색 (대공 필터 적용)
  2. 몬스터 발견 → 추격 + attackRange 내 공격
  3. 몬스터 없음 + 포탈 사거리 내 → 포탈 공격 (플레이어 유닛만)
  4. 위 모두 해당 없음 → 전방 전진
유닛 종류전진 방향경계선
플레이어 유닛오른쪽 (+x)x = 0 (넘지 않음)
AI 유닛왼쪽 (-x)x = 0 (넘지 않음)
  • 물리 기반 이동: Rigidbody2D + CircleCollider2D
  • 타겟 탐색은 UnitController.ActivePlayerUnits / MonsterController.AllMonsters 정적 리스트 활용 (매 프레임 FindObjectsOfType 호출 없음)

  • 스티키 타겟팅: 현재 공격 중인 유닛 대상을 유지. 대상이 죽으면 가장 가까운 유닛으로 재탐색
  • 유닛 없으면 목표 기지로 직진, 도달 시 1초 간격 반복 공격
  • 분리 조향(Separation Steering)으로 무리 겹침 방지
  • 비행 몬스터: 대공 능력 없는 유닛/건물이 공격 불가

속성수치

엘리트 소환 임계값HP 40% 미만
색상HP 100% = 보라, HP 0% = 빨강 (Lerp 그라디언트)
조건효과
피격마다포탈 우측(x=+1)에서 일반 몬스터 즉시 소환

14. 증원 시스템 (TransferSystem)

  • 트리거: 몬스터 처치
  • 지연: 1.5초 후 처치된 진영 포탈 우측에서 동일 등급 몬스터 재소환
더 많이 처치할수록 더 많은 몬스터가 쏟아진다 — 방어 효율 vs 압박 증가의 딜레마.
